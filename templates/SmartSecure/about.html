About SmartSecure

Special Thanks: We are deeply indebted to Professor Xiao Su, Professor Dan Harkey and Professor Chandrasekar Vupalapatti for their invaluable comments and assistance in the preparation of this project.
In this project, we aimed to build a composite framework or app for Android phones, that will watch user behavior, read patterns and be able to flag suspicious activity. Our approach is to use machine learning and data analytics to record how the users use their phone, the apps most used, the geographical locations the users likely visits and to flag any activity that is deemed to be outside normal. The scope of this project also includes checking the authenticity of Wi-Fi / Bluetooth connections. The outcome is a secure unified app or framework that actively protects and monitors the device and data in multiple ways. The high–level architecture of this project consists of the SmartSecure app (the frontend and app backend), the Django server and the database. Mobile Component Frontend: SmartSecure’s frontend is a limited number of screens that allow the user to and change various settings. The UI also includes the popup notifications and security questions that the app generates when it detects abnormal activity on the phone.
App backend: The backend accepts the user behavior model from the server. It performs the following tasks:
o Works in the background to collect user data
o Stores the data in its own local database
o Compares it against the model
o Evolves the model as per new statistics
o Pushes the collected data to the server at regular time intervals
o An Android service that runs continuously to monitor data such as the app usage data, Wi-Fi connectivity details and location data.
o A synchronization service that sends data to and receives data from the cloud server at regular intervals
o A database to temporarily store collected data and permanently store the model received from the cloud server
o A comparison service to compare incoming data with the model.
o A rules based engine to generate proper notifications
SmartSecure database: We are using MongoDb for SmartSecure’s data store.
Web component
The server follows the MVC pattern. It has APIs for the user as well as SmartSecure. It interfaces with MongoLabr on the backend.The serveris deployed on an AWS EC2 instances.
Frontend : The frontend will comprise webpages to show a dashboard with multiple charts to display app usage statistics
Backend : We are using Django framework to interface with Python APIs for our data processing needs. The backend is written in Python and data is passed among components as JSON.
